&cp_object; v1 Package Description

&cp_businessObjectClass; (&cp_object;) consists of many files all packaged here for you. This is a simple guide to the layout of this package. Before attempting to follow the links in this guide, please ensure that you have extracted the files from the zip file; otherwise, the links will not work.

As this is the standalone package, none of the dependant specs will be included and you will have to assemble the full package with an acceptable version of the dependent specs that you wish to use.

  • DES

    The Data Encoding Specification. This includes the DES document for this specification and the applicable signature memo associated with it.

    • ICO-NTK version 1
      • DesProfileICO-NTKXml.pdf
      • SignatureMemo.pdf

  • Examples

    Collection of example instance documents for this specification.

    • ICO-NTK

  • Schematron

    Collection of constraint rules applicable to this DES. These rules depend on the CVEs being located at the "correct" relative path they are in this package. Many of the dependent DES have applicable constraint rules; each one is documented separately. Each DES has 3 main files.

    • xxx_Rules.pdf

      A generated PDF collection of all the constraint rules. They are actually many individual .sch files. This file is a more convenient format to read but is not useful for execution of the rules.

    • xxx_XML.sch

      The top level .sch file that includes all of the others for the particular DES.

    • xxx_XML.xsl

      A pre "compiled" version of the .sch file ready for execution with any XSLT2 capable engine.

    Applicable constraint rules are as follows:

    • ICO-NTK
      • ICO-NTK_Rules.pdf
      • ICO-NTK_XML.sch
      • ICO-NTK_XML.xsl

  • manifest.md5

    This file contains an MD5 checksum of every file in the package. It can be used to determine if any of the files were corrupted in unzipping or if you have modified a file.

  • ICO-NTKpackage.properties

    This file contains information concerning the version, classification, and build time for the package.
